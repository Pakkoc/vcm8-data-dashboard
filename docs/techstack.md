## 대학교 데이터 시각화 대시보드 MVP 개발 기술 스택

프로젝트의 목표와 요구사항을 면밀히 재검토한 결과를 바탕으로, MVP(최소 기능 제품)를 가장 신속하고 정확하게 완성하기 위한 최종 기술 스택을 제안합니다.

### 최종 추천 기술 스택 요약

| 구분 | 기술 스택 | 핵심 역할 및 선정 이유 |
| :--- | :--- | :--- |
| **백엔드** | **Python (Django + DRF)** | **Problem Solver:** `Pandas`를 이용한 압도적으로 간결한 엑셀 데이터 처리, 내장 관리자 기능으로 개발 속도 극대화 |
| **프론트엔드** | **React (Vite + Recharts)** | **Visual Expert:** 가장 풍부한 생태계를 가진 표준 UI 라이브러리. 안정적이고 미려한 차트/그래프 구현에 최적화 |
| **데이터베이스 & 인증** | **Supabase (PostgreSQL)** | **Instant Foundation:** 클릭 몇 번으로 DB 구축 완료. 로그인 기능까지 내장되어 있어 구현 시간 '0'에 수렴 |
| **배포** | **Railway** | **Launchpad:** GitHub 연동을 통한 원클릭 배포. 인프라 구축 및 관리에 드는 모든 시간을 제거 |

---

### 왜 이 조합이 최선인가?

저희의 목표는 완벽한 시스템이 아닌, **가장 빠르게 핵심 가치를 검증할 수 있는 프로토타입**을 만드는 것입니다. 이 관점에서 각 기술은 다음과 같은 명확한 역할을 수행합니다.

1.  **백엔드 (Django): 핵심 문제를 가장 빨리 해결합니다.**
    프로젝트의 가장 큰 허들은 '엑셀 데이터를 읽어 DB에 저장'하는 것입니다. Python의 `Pandas` 라이브러리는 이 작업을 단 몇 줄의 코드로 가장 안정적으로 처리합니다. Django의 내장 관리자 기능은 개발자가 데이터를 즉시 확인하고 검증할 수 있는 강력한 도구를 무료로 제공하여 개발 속도를 비약적으로 향상시킵니다.

2.  **프론트엔드 (React): 시각화 요구사항을 가장 확실하게 충족합니다.**
    다양한 그래프와 차트를 구현해야 하는 요구사항에 있어, React는 가장 방대한 레퍼런스와 라이브러리 생태계를 보유한 가장 안전한 선택지입니다. `Recharts`와 같은 검증된 라이브러리를 통해 요구사항에 맞는 모든 시각화 결과물을 빠르고 직관적으로 만들어낼 수 있습니다.

3.  **DB & 배포 (Supabase, Railway): 인프라 고민을 완벽하게 제거합니다.**
    MVP 단계에서 서버를 세팅하고, DB를 설치하며, 배포 파이프라인을 구축하는 것은 가장 큰 시간 낭비입니다. Supabase는 DB와 로그인 기능을 즉시 제공하고, Railway는 코드를 GitHub에 올리는 것만으로 배포를 완료시킵니다. 이를 통해 저희는 오직 비즈니스 로직 개발에만 100% 집중할 수 있습니다.

### 결론

이 기술 스택은 단순히 인기 있는 기술의 조합이 아닙니다. **백엔드에서 가장 어려운 문제를 해결하고, 프론트엔드에서 가장 중요한 결과물을 보여주며, 인프라는 그 모든 과정을 투명하게 지원**하는 최적의 워크플로우를 구성합니다.

이 제안을 통해 귀교의 데이터를 가장 빠르고 효율적으로 시각화하여, 내부 베타테스트를 성공적으로 시작할 수 있을 것이라 확신합니다.
