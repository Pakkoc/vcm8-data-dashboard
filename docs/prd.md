## **제품 요구사항 정의서(PRD): 대학교 데이터 시각화 대시보드 MVP**

- **작성일:** 2025년 11월 13일
- **작성자:** CTO

### 1. 제품 개요 (Product Overview)

본 문서는 대학교 내부 데이터(실적, 논문, 학생 수, 예산 등)의 시각적 분석을 위한 **웹 기반 대시보드 MVP(Minimum Viable Product)** 개발 요구사항을 정의합니다.

#### 1.1. 문제 정의 (Problem)

현재 대학 내부 데이터는 이카운트(Ecount) 시스템을 통해 관리되고 있으나, 기본 제공되는 그래프의 시인성과 활용도가 낮아 데이터를 통한 직관적인 현황 파악 및 의사결정에 어려움을 겪고 있습니다.

#### 1.2. 해결 방안 (Solution)

이카운트에서 추출한 엑셀(Excel) 데이터를 **관리자가 직접 업로드하여**, 주요 성과 지표를 **한눈에 파악할 수 있는 맞춤형 시각화 대시보드**를 구축합니다. 이를 통해 데이터 기반의 신속하고 정확한 의사결정을 지원합니다.

#### 1.3. MVP 목표 (MVP Goal)

가장 핵심적인 기능인 '엑셀 업로드'와 '주요 지표 시각화'를 가장 빠른 방법으로 구현하여, 내부 직원을 대상으로 베타테스트를 진행하고 실제 사용성을 검증하는 것을 목표로 합니다. 완벽한 시스템이 아닌, **핵심 가치를 가장 빠르게 검증할 수 있는 프로토타입**을 만드는 데 집중합니다.

>  이를 위해 부가적인 편의 기능(예: 업로드 내역 관리, 데이터 자동 연동)은 다음 단계로 이연하고, 명시된 핵심 기능 구현에만 모든 리소스를 집중합니다.

####  1.4. MVP 범위에서 명시적으로 제외되는 기능 (Out of Scope)

프로젝트의 속도와 핵심 가치 검증에 집중하기 위해, 아래 기능들은 이번 MVP 범위에서 **명확하게 제외**합니다.

1.  **데이터 자동 연동 기능:**
    *   특정 폴더를 감시(watching)하여 파일이 추가될 때 자동으로 데이터를 동기화하는 기능은 구현하지 않습니다. 이는 MVP 단계의 복잡성을 가중시키는 오버엔지니어링입니다.
2.  **데이터 업로드 내역(History) 관리 기능:**
    *   과거에 업로드한 파일의 목록, 성공/실패 여부, 롤백 등의 기능은 포함하지 않습니다. 데이터 검증은 관리자가 업로드 후 메인 대시보드의 차트가 정상적으로 변경되었는지 직접 확인하는 것으로 충분합니다.

### 2. Stakeholders

| 역할 | 이름/팀 | 주요 책임 및 기대사항 |
| :--- | :--- | :--- |

### 3. 포함 페이지 (Scope of Pages)

MVP 범위 내에서는 아래의 3개 핵심 페이지만을 개발합니다.

1.  **로그인 페이지 (Login Page)**: 허가된 사용자만 접근할 수 있는 인증 화면
2.  **데이터 업로드 페이지 (Data Upload Page)**: 관리자가 엑셀 파일을 업로드하는 관리자 전용 화면
3.  **메인 대시보드 페이지 (Main Dashboard Page)**: 핵심 지표가 시각화된 데이터를 조회하는 메인 화면

### 4. 사용자 여정 (User Journey)

#### 4.1. 타겟 유저 Segment

*   **Segment A: 관리자 (Admin)**
    *   **정의:** 이카운트 시스템에서 데이터를 추출하고, 대시보드 시스템에 데이터를 업로드할 책임이 있는 실무 담당자.
    *   **목표:** 복잡한 과정 없이 엑셀 파일을 시스템에 업로드하여 최신 데이터로 대시보드를 갱신하는 것.

*   **Segment B: 일반 사용자 (General User)**
    *   **정의:** 대학의 주요 의사결정자(총장, 학과장 등) 및 성과 지표에 관심 있는 내부 교직원.
    *   **목표:** 언제 어디서든 웹을 통해 최신 성과 지표를 직관적인 그래프로 확인하고 인사이트를 얻는 것.

#### 4.2. 페이지별 사용자 여정

**A. 관리자 (Admin) 여정**

1.  **(로그인 페이지)**: 부여받은 아이디와 비밀번호를 입력하여 시스템에 로그인합니다.
2.  **(메인 대시보드 페이지)**: 로그인 후 가장 먼저 보게 되는 메인 대시보드에서 현재 데이터를 확인합니다.
3.  **(데이터 업로드 페이지)**: 헤더(GNB)의 '데이터 관리' 메뉴를 통해 데이터 업로드 페이지로 이동합니다.
4.  **(데이터 업로드 페이지)**: '파일 선택' 버튼을 클릭하여 로컬에 저장된 이카운트 엑셀 파일을 선택하고 '업로드' 버튼을 누릅니다.
5.  **(데이터 업로드 페이지)**: 시스템이 파일 파싱 및 저장을 완료하면 "성공적으로 업로드되었습니다" 라는 피드백 메시지를 확인합니다. (오류 발생 시, "데이터 형식이 올바르지 않습니다" 와 같은 메시지 확인)
6.  **(메인 대시보드 페이지)**: '대시보드' 메뉴를 통해 메인 페이지로 돌아와, 방금 업로드한 데이터가 그래프에 정상적으로 반영되었는지 눈으로 검증합니다.

**B. 일반 사용자 (General User) 여정**

1.  **(로그인 페이지)**: 부여받은 아이디와 비밀번호를 입력하여 시스템에 로그인합니다.
2.  **(메인 대시보드 페이지)**: 로그인 후 표시되는 메인 대시보드에서 실적, 논문, 학생 수, 예산 등 핵심 지표를 그래프(막대, 라인, 파이 차트 등)로 확인합니다.
3.  **(메인 대시보드 페이지)**: 특정 지표의 시간별 추이(라인 차트)나 항목별 비중(파이 차트)을 보며 성과를 직관적으로 파악합니다.
4.  **(메인 대시보드 페이지)**: 필요한 정보 확인 후, 헤더의 '로그아웃' 버튼을 눌러 세션을 종료합니다.

### 5. 정보 구조 (Information Architecture)

MVP의 정보 구조는 사용자가 최소한의 탐색으로 목표를 달성할 수 있도록 단순하게 설계합니다.

```
[대학교 데이터 시각화 대시보드]
 └── / (Root)
     ├── 로그인 페이지 (Public Route)
     │
     └── (인증 후 접근 가능한 Private Routes)
         ├── 메인 대시보드 페이지 (Default)
         │   ├── 1. 종합 실적 현황 (막대 그래프)
         │   ├── 2. 논문 게재 수 추이 (라인 차트)
         │   ├── 3. 재학생/신입생 비율 (파이 차트)
         │   └── 4. 예산 집행률 (게이지 차트 등)
         │
         └── 데이터 업로드 페이지 (Admin 전용)
             ├── 1. 엑셀 파일 업로드 컴포넌트
             │
             └── 업로드 내역/히스토리 기능은 MVP 범위에서 명시적으로 제외
```
